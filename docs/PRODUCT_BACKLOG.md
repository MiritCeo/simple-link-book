# Backlog pomysłów na rozwój produktu

Zbiór pomysłów na funkcje i moduły **bez zobowiązania do wdrożenia** — żeby nic nie uciekło.  
Status domyślny: **pomysł / backlog**. Priorytety i sprinty ustalacie osobno.

**Jak dopisywać:** dodaj wiersz w odpowiedniej sekcji albo nową sekcję; krótki opis + ewentualnie „dlaczego” w 1 zdaniu.

---

## Salon (panel B2B — właściciele / personel)

| ID   | Pomysł | Notatka |
|------|--------|---------|
| S-01 | Marketing z kalendarza | Segmenty (np. brak wizyty od X mies., pierwsza wizyta, jedna usługa) + prosta kampania SMS/e-mail z szablonem i linkiem do rezerwacji |
| S-02 | Program poleceń / kody rabatowe | Kod po wizycie dla znajomego, limity użyć, raport w panelu |
| S-03 | Mini-raporty biznesowe | Tygodniowy mail: obłożenie, no-show, przychód wg usług, top pracownicy |
| S-04 | Eksport / integracje księgowe | CSV, webhooki — decyzja zakupowa dla większych salonów |
| S-05 | Checklisty dnia / protokoły | Powiązanie z magazynem (np. sterylizacja, stany) |
| S-06 | Widok „wypełnij lukę” | Sloty z niskim obłożeniem + sugerowana akcja do segmentu klientów |

## Klienci (aplikacja / rezerwacja B2C)

| ID   | Pomysł | Notatka |
|------|--------|---------|
| C-01 | Ulubione usługi + przypomnienie „czas na wizytę” | Na podstawie interwału usługi (np. farba co N tygodni), szybka rezerwacja jednym tapnięciem |
| C-02 | Historia / „karta looku” | Zdjęcia przed/po, notatki widoczne przy kolejnej wizycie (beauty) |
| C-03 | Opinie po zweryfikowanej wizycie | Gwiazdki + komentarz po zakończonej wizycie; widoczne przy bookingu |
| C-04 | Rezerwacje grupowe / znajomi | Wspólny slot, „zarezerwuj dla siebie i osoby towarzyszącej” |
| C-05 | Push o promocjach tylko od ulubionych salonów | Preferencje częstotliwości, żeby uniknąć spamu |
| C-06 | Zaliczka / pełna przedpłata | Ograniczenie no-show; opcjonalnie tylko wybrane usługi lub nowi klienci |
